mobile first responsive web design

What is it?

Chosing to follow a mobile first approach makes you design for a mobile view first and then adapt the design for larger viewports in media queries rather then the opposite.

Responsive on the other hand is a uniform design that adapts to the viewport.

Combined they make a more easily maintained code base.

Features of this design

  • An easy to generate HTML markup for backend generated content.
  • First article becomes fullscreen for easy promotion
  • CSS only menu for medium to small viewports.
  • Fixed top bar with logo and menu
  • Optional fixed footer
  • Dual and tripple column layout at larger viewports
  • images auto resize to layout width

I started writing this design as a trial and error learning to make something not terrible. This is the third iteration of the same design, the other two failed. A fixed responsive top menu and bottom footer. I wanted a CSS-only base design and add JavaScript only if necessary in order to properly learn the ins and outs of responsive CSS through media queries and relative units.

When I was happy with the menu I started working on a full screen responsive article and researching the "vw" and "vh" units. It seems support is a bit wobbly still, even my IDE Sublime Text 2 doesn't know what it is.

With a full screen responsive article and menu I continued making a responsive collection of older articles. Low resolution yields one column, higher shows two and higher still gives you three columns. Im pretty happy with this layout. Now to make some content for it...

Wordpress integration.

Published: 2014-12-18 10:33:15 Modified: 2014-12-20 20:12:00 Written by: Tobias Axelsson

Say what you like about wordpress. I think it is a good enough place to write your blog on. What if you could write your posts on Wordpress and then have then show up on your site as they are published?

Lets see if we can do this. Wordpress apperently has an API for accessing posts of a particular site/author/blog/whatnot. I write my posts on the site: https://fend14tobis.wordpress.com You can access a post from this collection like so: 

https://public-api.wordpress.com/rest/v1/sites/fend14tobis.wordpress.com/posts/

Making a request for this data gives a collection of my posts in JSON-format. More to read here: https://developer.wordpress.com/docs/api/. Displaying it on another site will require either JavaScript for clientside access or serverside scripting like PHP or .NET. I'm gonna opt for JavaScript just for fun. There are a number of complications to this choice but it's all for the learning process.

  • Disabled JavaScript will not fetch content
  • Network may prove unreliable to fetch content
  • The content will not load untill the browser has parsed the script.
  • Constructing the elements in Javascript is not fun.
  • JavaScript should enhance content viewing, not allow content viewing.

Let's not think about those points. A request to the API is just an xhr away. Parse the response and construct the elements to contain the data. Add the elements to the DOM and we're done.
